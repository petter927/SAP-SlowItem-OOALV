# SAP 倉庫呆料分析 (OO-ALV) - ABAP 練習作品 👋

本專案是我依據倉儲與物料管理情境，實作的 SAP 呆滯庫存分析報表。  
透過工廠/倉庫/料號條件與呆滯天數門檻，快速找出高風險物料，協助庫存管理與採購決策。

---

## 🛠 實作重點與技術細節 (Implementation Highlights)

### 1. 多資料來源整合，建立可分析的庫存視圖
- **主檔與庫存整合**：由 `MARD` + `MARA` + `MAKT` 取得庫存、單位與物料描述。
- **幣別與估值補充**：串接 `T001K` / `T001` 補齊公司幣別資訊，方便金額呈現。
- **價格邏輯**：由 `MBEW` 依 `VPRSV` 判斷採用標準價或移動平均價，並依 `PEINH` 換算呆滯金額。

### 2. 呆滯判定演算法
- **交易紀錄分析**：讀取 `MSEG`，依移動類型分辨「最後入庫日」與「最後領用日」。
- **參考日期決策**：以「最後領用日 > 最後入庫日 > 建立日」的優先順序計算基準日期。
- **呆滯天數計算**：以 `sy-datum - ref_date` 推算 `slow_days`，可用參數 `p_days` 篩選目標物料。
- **風險燈號分級**：以綠/黃/紅燈圖示呈現狀態（例如 180 天、365 天分級），提高報表可讀性。

### 3. OO ALV 報表呈現與操作性
- **物件導向封裝**：以 `lcl_report` 將資料取得、商業邏輯與顯示層拆分成獨立 methods。
- **SALV 互動功能**：啟用排序、欄寬最佳化、斑馬紋、標準工具列功能，提升使用體驗。
- **欄位中文化**：統一設定 ALV 欄位標題，讓非技術使用者也能快速理解報表資訊。
- **關鍵數據彙總**：對呆滯金額與庫存數量提供 aggregation，便於管理層快速檢視。

---

## 📂 技術棧 (Tech Stack)
- **Language**: ABAP (7.40+ 語法風格)
- **Platform**: SAP ERP (SAP GUI Report)
- **Data Sources**: `MARD`, `MARA`, `MAKT`, `MBEW`, `MSEG`, `T001K`, `T001`
- **UI / Framework**: `CL_SALV_TABLE` (OO ALV)
- **Tools**: SAP GUI, ABAP Workbench (SE38/SE80)

---

## 💡 開發初衷
在工廠端，呆料不只是庫存問題，也直接影響資金占用與採購策略。  
這個專案讓我把「資料查詢」提升成「可判讀的分析報表」：除了列出庫存，還結合價格、最後異動紀錄與風險分級，讓使用者能更快找到需要優先處理的物料。
